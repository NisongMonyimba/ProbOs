Deterministic mode: inputs are fixed at their nominal values, so the model returns exactly one number. There is no way to know how often real cells — with real manufacturing tolerance, real thermal variation — would exceed a safety threshold.

Validated against: Kim et al. (2007), accelerating rate calorimetry data. Sobol sensitivity identifies SEI decomposition activation energy as the dominant risk driver (S1=0.457). This demo uses a simplified illustrative formula, not the real validated kernel — see GitHub for the actual model.

C++/OpenMP wins at every particle count tested — GPU kernel fusion genuinely improved GPU performance (up to 7.6x faster at small N versus the unfused version), and GPU now beats plain Python at N>=2,000, but reported honestly: it does not beat C++ on this hardware. We built and shipped this result rather than quietly dropping the GPU path when it didn't win.

PDSL v0.2 — declare a model, don't hand-write it
A small domain-specific language, now with control flow (Month 3 Week 13)
// A thresholded drift, compiling to vectorised numpy code model battery { state T1 = 298.0 param Ea_SEI ~ Normal(1.35e5, 5e3) drift T1 = if T1 > 403.15 then arrhenius(Ea_SEI, T1) else 0.0 }

Compiles to real, vectorised np.where(...) — verified with a genuine per-particle divergence proof (different simulated cells taking different branches in the same batch), not just "it compiles."

This is a toy model. The validated one is on GitHub. Battery, pharma, and automotive safety UQ — open source, Apache 2.0.
